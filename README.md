# AI 自动调参串口助手

这是一个 AI 自动调参的串口助手。它运行在电脑端,用于调试单片机里的 PID 控制器,支持实时曲线可视化调参、串口通信、参数预设保存/加载、曲线数据导出,以及基于**继电反馈 + Ziegler-Nichols** 的一键自动整定。内置虚拟被控对象,**无需任何硬件即可开箱试用**。

## 功能特点

- **可视化调参**:实时绘制设定值 / 测量值 / 输出三条曲线,响应好坏一目了然
- **自动整定**:一键激起系统极限环振荡,自动测出特性并算出 Kp/Ki/Kd
- **参数预设**:可把常用 Kp/Ki/Kd、目标值和自整定参数保存成 JSON,下次直接加载
- **数据导出**:可将实时曲线导出为 CSV,便于用 Excel / Python 继续分析
- **无硬件演示**:内置二阶惯性 + 纯滞后仿真对象,默认虚拟仿真模式,打开即用
- **真实硬件无缝切换**:界面里把连接模式切到"真实串口"就行,底层用的是同一套文本通信协议。你的单片机固件只要能收发这几条简单指令(设参数、设目标、切模式、直接给输出、复位),用 `scanf`/`sprintf` 就能实现,零依赖

## 技术构成

- **界面**:Python + PySide6(Qt 桌面框架)+ pyqtgraph(高性能实时曲线)
- **通信**:串口 UART,文本行协议
- **架构分三层**:
  - 纯算法层(PID、仿真对象、自整定,不依赖界面,可单独测试)
  - 通信层(串口抽象,真实 / 虚拟两种实现)
  - 界面层(后台采集线程 + 主窗口,曲线刷新和串口读写分开,界面不卡)

## 运行

```bash
pip install -r requirements.txt
python main.py
```

窗口打开后连接模式默认是**虚拟仿真**,不接硬件即可试:

1. 点**连接** → 内部启动一个仿真单片机(PID + 二阶被控对象)
2. **手动调参**:填 Kp/Ki/Kd 和目标值 → 下发参数 → 启动,观察曲线响应
3. **自动整定**:点开始自整定 → 继电反馈自动激起振荡 → 几秒后算出 Kp/Ki/Kd 自动填入 → 下发并启动验证
4. **记录结果**:调好后可保存参数预设,也可导出 CSV 曲线做复盘

## 打包为桌面 exe

```bash
pip install pyinstaller
pyinstaller --noconfirm PIDTuner.spec
```

产物为 `dist/PIDTuner.exe`,单文件、双击即用、不依赖 Python 环境。

> `dist/PIDTuner.exe` 体积较大,建议在 Git 仓库中使用 Git LFS 管理。

## 通信协议(接真实硬件时)

界面里连接模式切到**真实串口**,选端口和波特率即可。单片机固件只需实现这个文本行协议:

**上行**(下位机 → 上位机,每个采样点发一行):

```
D,<t_ms>,<setpoint>,<measurement>,<output>\n
```

**下行命令**(上位机 → 下位机):

| 命令 | 格式 | 含义 |
|------|------|------|
| 设参数 | `S,<kp>,<ki>,<kd>\n` | 设置 PID 参数 |
| 设目标 | `T,<setpoint>\n` | 设置目标值 |
| 切模式 | `M,<0\|1\|2>\n` | 0=停止 / 1=PID 运行 / 2=直接输出 |
| 直接给输出 | `O,<u>\n` | 直接给定输出量(自整定用) |
| 复位 | `R\n` | 复位 / 清零 |

单片机端用 `scanf`/`sprintf` 即可解析,零依赖。

## 说明

- 自整定用经典 Ziegler-Nichols,`classic_pid` 规则偏激进(约 30%+ 超调)。想要平稳可选 `no_overshoot` 或 `some_overshoot`。
- 单文件 exe 首次启动需解压,约 2-3 秒;偶尔被杀软误报,加白名单即可。
